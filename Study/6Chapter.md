# Git 챕터 6: 원격 저장소 연동 (Clone, Push, Pull, Fetch)

## 1. 원격 저장소 복제 (Clone)

**개념:** 원격 저장소(예: GitHub)의 모든 파일과 커밋 이력을 지역 저장소로 통째로 복제하는 작업입니다.

| 명령 | 역할 및 설명 |
| :--- | :--- |
| **`$ git clone [원격 저장소 URL]`** | 원격 저장소의 내용을 현재 폴더에 복사하고, 자동으로 로컬 저장소를 초기화합니다. |
| **`$ git remote`** | 원격 저장소의 별칭(alias) 목록을 확인합니다. (기본값: **origin**) |
| **`$ git remote -v`** | 원격 저장소의 별칭과 함께 Fetch/Push URL을 자세히 표시합니다. |
| **`$ git remote show [별칭]`** | 원격 저장소의 이름, HEAD, 로컬 브랜치의 추적 정보 등을 상세하게 표시합니다. |

## 3. 지역 저장소 반영 (Push)

**개념:** 지역 저장소에서 새로 커밋한 이력(버전)을 원격 저장소로 **업로드**하여 반영하는 작업입니다.

| 명령 | 역할 및 설명 |
| :--- | :--- |
| **`$ git push [별칭] [로컬 브랜치]`** | 로컬 브랜치의 변경 이력을 원격 저장소의 해당 브랜치로 전송합니다. |
| **`$ git push origin main`** | 지역의 `main` 브랜치를 원격의 `origin`으로 푸시합니다. |
| **`$ git push -u origin main`** | **최초 푸시 시** 로컬 `main`이 원격 `origin/main`을 추적하도록 설정합니다. (이후부터는 `$ git push`만 사용 가능) |
| **`$ git branch -vv`** | 로컬 브랜치가 추적하는 원격 브랜치 정보를 확인합니다. |

## 4. 원격 저장소 반영 가져오기 (Pull vs Fetch)

원격 저장소의 변경 이력을 지역 저장소로 가져오는 방법에는 `pull`과 `fetch` 두 가지가 있습니다.

### 4.1. Pull (가져오기 + 자동 병합)

| 명령 | 역할 및 설명 |
| :--- | :--- |
| **`$ git pull [별칭] [브랜치명]`** | 원격 저장소의 변경 이력을 가져와 (Fetch) **바로 지역 브랜치에 병합(Merge)**까지 한 번에 처리합니다. |
| **`$ git pull origin main`** | 원격 `origin`의 `main` 브랜치 내용을 가져와 로컬 `main`에 병합합니다. |
| **특징:** **빠르지만** 충돌(Conflict)이 발생하면 즉시 해결해야 합니다. |

### 4.2. Fetch (가져오기만)

| 명령 | 역할 및 설명 |
| :--- | :--- |
| **`$ git fetch [별칭] [브랜치명]`** | 원격 저장소의 변경 이력(데이터)만 가져와 **원격 추적 브랜치**(`origin/main` 등)를 업데이트합니다. **지역 브랜치는 변경되지 않습니다.** |
| **`$ git fetch origin`** | 원격 `origin`의 모든 브랜치 정보를 가져옵니다. |
| **`$ git log --oneline --all`** | Fetch 후 원격 추적 브랜치(`origin/main`)에 새로운 커밋이 생긴 것을 로그로 확인할 수 있습니다. |
| **병합 (Merge):** Fetch 후 **`$ git merge origin/main`** 명령을 사용하여 수동으로 병합을 수행합니다. |
| **장점:** 변경 사항을 **미리 확인**한 후 안전하게 병합할 수 있습니다. |
