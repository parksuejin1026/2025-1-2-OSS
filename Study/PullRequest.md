## 📄 Pull Request (PR) 정리

Pull Request(PR)는 수정 권한이 없는 타인의 저장소에 자신의 기여를 반영해 달라고 요청하는 알림입니다.

### 1. Git/GitHub 기본 개념

* **Push & Pull (지역 저장소 관점):**
    * **Push:** 지역 저장소의 변경 사항을 **원격 저장소(Remote)**로 올리는 작업입니다.
    * **Pull:** 원격 저장소의 변경 사항을 **지역 저장소(Local)**로 내리는 작업입니다.
    * 원격 저장소는 GitHub 서버에 위치하며, **권한이 있는 사용자**만 직접 Push가 가능합니다.
* **Origin vs Upstream (원격 저장소의 종류):**
    * **Upstream:** **오픈소스(프로젝트)** 또는 타인의 저장소로, 프로젝트의 **원천**이 되는 저장소입니다.
    * **Origin (Remote):** **Upstream**을 내 계정으로 **Fork**한 자신의 저장소입니다.
    * **Fork**는 원본 프로젝트에 영향을 주지 않고 자유롭게 변경 사항을 실험할 수 있게 복사본을 만드는 것입니다.

### 2. Pull Request (PR) 정의 및 과정

* **PR 정의:** 수정 권한이 없는 타인의 저장소에 저장소 수정 등의 기여 요청을 하는 알림입니다.
    * 이는 기여자의 수정 내용을 가져와(**pull**) 소유자의 저장소에 **병합(merge)** 하는 작업을 요청하는 것입니다.
* **PR 작성자 (Developer):** Upstream을 Fork하거나 직접 수정해서 PR을 요청할 수 있습니다.
* **PR 처리 과정:**
    1.  **Fork** (New Branch): 기여자가 Upstream을 복제합니다.
    2.  **Discussion, develop** (개발 및 논의): 코드를 수정하고 작업합니다.
    3.  **Commit:** 변경 사항을 커밋합니다.
    4.  **Pull request:** Upstream 저장소에 수정 요청을 보냅니다.
    5.  **Request review & discussion:** 관리자가 PR을 검토하고 논의합니다.
    6.  **Accept PR & Merge:** 관리자가 승인 후 병합(Merge)을 수행합니다.
    7.  **Sync Fork:** PR이 병합된 후에도 기여자의 Remote 저장소는 아직 동기화되지 않은 상태이므로, **Sync fork** > **Update branch**를 통해 Upstream과 동일하게 만듭니다.

## 🛠️ 추가 수업 내용 통합 (Git 심화 명령)

### 1. 버전 되돌리기 (`git reset`) 심화 (Chapter 10)

`git reset`은 특정 커밋 이후의 이력을 **삭제**하는 명령어입니다.

* **`--hard`:** 지정 커밋의 내용으로 **모두 복사**합니다. 작업 디렉토리, 스테이징 영역, 깃 저장소(**이후 이력**)의 변경 사항을 **모두 삭제**합니다.
* **`--mixed` (기본):** 깃 저장소(**이후 이력**)는 삭제되지만, 작업 디렉토리의 변경 사항은 **유지**됩니다.
* **`--soft`:** 깃 저장소(**이후 이력**)만 삭제하고, 변경 사항은 작업 디렉토리와 스테이징 영역에 **모두 유지**됩니다.

### 2. 커밋 취소 (`git revert`) 활용 (Chapter 10)

* **`$ git revert HEAD --no-edit`:**
    * 가장 최근 커밋의 변경 사항을 되돌리는 **새로운 커밋(취소 커밋)**을 생성합니다.
    * 이전 커밋 히스토리가 **그대로 유지**되므로, **원격 저장소에 Push 한 이력**을 되돌릴 때 안전하게 사용됩니다.

### 3. Git Stash 옵션 (Chapter 7)

* **`$ git stash apply --index`:**
    * 임시 저장된 내용을 가져와 반영할 때 **스테이징 영역 상태까지 그대로 복원**하고, 스태시 목록은 **삭제되지 않습니다**.

### 4. Git Alias 설정 (Chapter 2)

* **`$ git config --global alias.lg1 'log --oneline'`:**
    * `$ git lg1`을 입력하면 `$ git log --oneline`과 동일하게 실행되도록 별칭(**Alias**)을 설정하는 명령어입니다.
