📄 교과목 OSS 수업 준비 자료 요약
본 자료는 동양미래대학교 인공지능소프트웨어학과 강환수 교수의 

오픈 소스 소프트웨어(OSS) 수업을 준비하기 위한 기초 안내서입니다.



1. 수업 준비 단계 및 필수 환경 설정
수업을 시작하기 위해 수강생들은 몇 가지 준비 작업을 수행해야 합니다. 먼저, 

반 대표를 선정하고 팀을 편성합니다(A반, B반 각 5~6명). 기술적인 준비로는 분산 버전관리 시스템인 

Git과 코드 편집기인 VSCode를 설치해야 합니다. 또한, 소프트웨어 협업 개발을 위한 웹 호스팅 서비스인 

GitHub.com에 접속하여 계정을 생성하고 로그인하는 과정이 필수적입니다.

2. 필수 소프트웨어 이해
이 수업에서 다루는 주요 소프트웨어는 다음과 같습니다.


Git (깃): 소스코드 관리를 위한 분산 버전관리 시스템입니다. 무료이며 오픈 소스로 제공됩니다. 공식 홈페이지는 



https://git-scm.com/이며, Git 자체의 소스 코드는 GitHub에서 https://github.com/git/git 주소로 확인할 수 있습니다.



GitHub (깃허브): Git 기반의 저장소 및 소프트웨어 협업 개발을 위한 웹 호스팅 서비스입니다. 가장 인기 있는 버전 관리 저장소 호스팅 서비스 중 하나이며, 영리 서비스와 오픈소스를 위한 무상 서비스를 모두 제공합니다.



VSCode (비주얼 스튜디오 코드): 마이크로소프트(MS)사가 개발하는 오픈 소스 코드 에디터 소프트웨어입니다. 가볍지만 강력한 소스 코드 편집기로, Windows, macOS, Linux 등 다양한 운영체제에서 사용 가능합니다. JavaScript, TypeScript 등을 기본적으로 지원하며, 


확장 기능(extension) 에코 시스템을 통해 C++, Python 등 다양한 언어를 지원합니다.


3. GitHub 계정 및 저장소(Repository) 사용법

GitHub 계정 생성은 이메일 준비 후 Github.com/signup 페이지에서 진행하며, 이메일, 암호, 사용자 이름을 입력하고 메일로 수신된 코드를 입력하는 과정을 거칩니다.





저장소 생성은 로그인 후 Create repository 버튼을 통해 이루어지며, 저장소 소유자(Owner)와 저장소 이름, 그리고 간략한 **설명(Description)**을 입력합니다. 한 계정 내에서 저장소 이름은 중복될 수 없습니다. 저장소는 **공개(Public) 또는 비공개(Private)**로 선택할 수 있습니다. 이 때, 저장소를 초기화할 때 **





Add a README file**을 체크하지 않으면 저장소에 브랜치가 하나도 없는 상황이 될 수 있습니다.




저장소 삭제는 저장소 페이지의 메뉴 **Settings**에 있는 **Danger Zone**에서 **Delete this repository**를 선택하여 진행합니다.



4. Fork 기능과 저장소 종류
**Fork (포크)**는 다른 깃허브 저장소(오픈 소스 프로젝트)를 

나의 계정으로 복사하는 작업입니다. 이를 통해 원본 프로젝트에 영향을 주지 않고 자유롭게 변경사항을 실험할 수 있습니다.


Fork와 관련된 저장소의 명칭은 다음과 같습니다:


upstream: 타인(Them)의 오픈 프로젝트 원본 저장소입니다.



origin: 나의 원격 저장소, 즉 GitHub에서 내 계정에 있는 Fork한 저장소를 말합니다.




local: 내가 자신의 컴퓨터에 복사한 지역 저장소입니다.


Fork된 저장소 동기화: 만약 원본 저장소(upstream)가 수정되어 내 저장소가 뒤처지게 되면(This branch is 2 commits behind ai7dnn:main과 같은 문구 표시), Sync fork 아래의 Update branch 버튼을 클릭하여 원본의 최신 내용을 내 저장소에 반영할 수 있습니다.
