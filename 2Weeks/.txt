Ⅰ. 깃(Git) 설치와 실행 (04. 2-1 깃 설치와 실행.pdf)
다운로드:

깃 공식 홈페이지(https://git-scm.com/)에서에서) 다운로드할 수 있습니다. 


홈페이지에서는 Pro Git이라는 책의 한국어 번역본도 제공하여 학습에 활용할 수 있습니다. 



설치:

설치 과정 중 기본 편집기를 'Visual Studio Code'로 설정하는 것이 권장됩니다. 


새 저장소(Repository)를 만들 때 기본 브랜치(Branch) 이름을 'main'으로 설정할 수 있습니다. 



실행:

윈도우 시작 메뉴에서 'Git Bash'를 검색하여 실행할 수 있습니다. 

원하는 폴더에서 마우스 오른쪽 버튼을 클릭하고 'Git Bash Here'를 선택하여 바로 실행할 수도 있습니다. 

파일 탐색기의 주소창에 'git bash'를 직접 입력하여 현재 위치에서 실행하는 방법도 있습니다. 

버전 확인:

Git Bash에서 $ git --version 명령어로 설치된 깃의 버전을 확인할 수 있습니다. 


Ⅱ. 깃(Git) 설정과 저장소 생성 (05. 2-2 [실습]깃 설정과 저장소 생성.pdf)

최초 설정: 깃을 설치한 후, 가장 먼저 사용자 이름과 이메일 주소를 설정해야 합니다. 


$ git config --global user.name "사용자 이름" 


$ git config --global user.email "이메일 주소" 

주요 설정 (저장소 생성 전 권장):


사용자 정보: 이름(user.name)과 이메일(user.email) 



줄바꿈 처리: 운영체제 간의 줄바꿈 문자 차이로 인한 문제를 방지합니다. 

윈도우에서는 core.autocrlf를 true로 설정합니다. 




기본 편집기: core.editor를 'code --wait' (VS Code) 등으로 설정합니다. 



기본 브랜치 이름: init.defaultBranch를 'main'으로 설정합니다. 


설정 범위:


--global: 현재 사용자의 모든 저장소에 적용됩니다. 





--local: 현재 저장소에만 적용됩니다. 




--system: 시스템의 모든 사용자에게 적용됩니다. 


저장소 생성:


$ git init: 현재 디렉토리를 깃 저장소로 만듭니다. 



$ git init [폴더 이름]: 지정한 이름의 하위 폴더를 생성하고 그곳을 깃 저장소로 만듭니다. 

저장소를 생성하면 .git이라는 숨김 폴더가 만들어지며, 이곳에 모든 버전 기록이 저장됩니다. 



Ⅲ. 비주얼 스튜디오 코드(VS Code)와 리눅스 명령어 (06. 2-3 비주얼스튜디오코드와 리눅스 명령.pdf)
비주얼 스튜디오 코드(VS Code) 개요:

마이크로소프트에서 개발한 소스 코드 편집기로, 가볍고 강력하며 Windows, macOS, Linux 등 다양한 운영체제에서 사용할 수 있습니다. 


'확장(Extensions)' 기능을 통해 다양한 언어와 개발 환경을 지원합니다. 


VS Code 설치:

공식 홈페이지(https://code.visualstudio.com/)에서에서) 다운로드할 수 있습니다. 

설치 시 "Code(으)로 열기" 옵션을 체크하면 탐색기에서 폴더나 파일을 바로 VS Code로 열 수 있어 편리합니다. 


주요 리눅스 명령어: Git Bash 환경에서 유용하게 사용되는 기본 명령어들입니다.


pwd: 현재 작업 중인 폴더(디렉토리)의 경로를 보여줍니다. 


ls: 현재 폴더의 파일 및 하위 폴더 목록을 보여줍니다. (-al 옵션을 사용하면 숨김 파일과 상세 정보를 함께 볼 수 있습니다.) 



cd [폴더 이름]: 지정한 폴더로 이동합니다. 


touch [파일 이름]: 비어있는 새 파일을 생성합니다. 


cat [파일 이름]: 파일의 내용을 터미널에 출력합니다. 


echo [내용] > [파일 이름]: 파일에 내용을 덮어쓰기 합니다. 



echo [내용] >> [파일 이름]: 파일의 끝에 내용을 추가합니다. 



rm [파일 이름]: 파일을 삭제합니다. 


mv [기존 이름] [새 이름]: 파일 또는 폴더의 이름을 변경합니다
